## 0. 포지셔닝 (모든 채널의 전제)

- 이 제품은 **계산기(시뮬레이터)** 다. **종목 추천·투자 자문이 아니다.** 카피·콘텐츠·커뮤니티
  어디서도 "이 종목을 사라 / 이 포트폴리오가 좋다"로 읽히면 안 된다. 항상 **"조건을 넣으면 계산해
  준다"** 프레임을 유지한다(규제 리스크 회피 + 제품 정체성). → §6 리스크.
- 차별화 축: 다른 배당 계산기는 "지금 배당이 얼마"를 답한다. 우리는 **"언제 월 OOO원에 도달하는가 /
  내 배당 목표 도달 나이"** 라는 **시간축·목표축**을 답한다. 이게 검색·공유 훅의 핵심이다(§5).

## 1. 목표와 지표 — 무엇을 "유입 성공"으로 보나

### North Star: 주간 활성 시뮬레이터 (WAS)

> **한 주간 `simulation_result_view`에 도달한 순 방문 수(신규 + 재방문).**

- **왜 이것인가**: 방문(page_view)은 이탈 포함 허수다. 이 제품의 가치는 **결과 화면에서 전달**되므로,
  "결과까지 온 방문"만이 진짜 획득이다. 이 지표는 **획득(신규가 결과까지 옴)** 과 **리텐션(재방문이
  다시 돌림)** 을 한 숫자로 묶는다. `simulation_result_view`는 이미 GA4 **주요 이벤트(Key Event)** 라
  즉시 측정 가능하다(ga4-plan §2-1).
- **왜 "가입"이 아닌가**: 로그인은 저장·커뮤니티에만 필요한 **선택적** 행동이다. 무로그인으로도 핵심
  가치를 다 쓸 수 있으므로 가입을 North Star로 삼으면 제품 철학과 어긋난다(가입은 리텐션 보조지표).

### 보조지표 (입력지표 / 가드레일)

| 지표 | 정의 (GA4) | 역할 |
|---|---|---|
| **활성화율** | `simulation_result_view` / 세션 | 유입 **품질**. 트래픽이 늘어도 이게 떨어지면 잘못된 검색어를 잡은 것 |
| **공유 생성 수** | `scenario_shared` 카운트 | **바이럴 루프**의 입력. 활성 유저 대비 공유율(K인자 근사) |
| **커뮤니티 글 수(누적)** | `community_post_published` 누적 | **SEO 콘텐츠 공급량** = 콜드스타트 해소의 직접 척도 |
| **색인 페이지 수** | Search Console / 네이버 서치어드바이저 | SEO **표면적**. ISR 인프라가 실제로 먹히는지 |
| **재방문율** | `is_returning` 코호트, `return_visit` | **리텐션**. 재방문이 늘면 WAS가 복리로 오른다 |
| **유입 채널 구성** | GA4 트래픽 획득(source/medium) | organic-search(구글/네이버) vs referral(카페·커뮤니티) vs direct(공유) 비중 |

### 콜드스타트 단계의 현실적 목표 (숫자는 가설)

> ⚠ 현재 GA4 실데이터는 콜드스타트라 사실상 0에 수렴한다. 아래는 **관측 후 보정할 가설 목표**이며,
> 30일 실측이 쌓이면 `analytics-analyst`가 baseline을 뽑아 재설정한다. **없는 숫자를 지어내지 않는다.**

- **D0~30**: 인프라·계측 완비 + baseline 확보. 색인 페이지 20+ (도구 랜딩 + 시드 글).
- **D30~60**: 주간 organic 세션 우상향 진입, WAS 주 두 자릿수, 공유 생성 첫 발생.
- **D60~90**: 롱테일 검색어 1~2개에서 첫 페이지 진입, 커뮤니티 글 30+, 외부 커뮤니티 referral 발생.

## 2. 채널별 전략

각 채널: **실행안 · 비용 · 난이도 · 예상효과 · GA4 검증**.

### 2-A. SEO / 콘텐츠 — *최우선(복리 자산)*

방금 배포한 **ISR 4단계**가 핵심 무기다: 크롤러가 **JS 없이** 게시글 본문·목록·메타를 읽고,
동적 `/api/sitemap`이 URL을 노출하며, `/api/og`가 공유 카드를 그린다. 특히 **네이버 Yeti·다음**은
JS를 렌더하지 않으므로, 서버 본문 주입이 없으면 SPA는 이들에게 **빈 페이지**다 — 이 인프라가 한국
시장 SEO의 전제다. 문제는 **색인할 콘텐츠가 없다는 것**(공개 글 3건). 그래서 SEO의 성패는 §2-C
커뮤니티 시딩과 한 몸이다.

**3개 콘텐츠 축:**

1. **도구 자체 랜딩 (transactional 검색어)** — "배당 계산기", "배당 재투자 계산기", "스노우볼
   계산기", "월배당 계산기". 검색 의도가 곧 우리 도구다. 메인(`/`)의 메타·H1·JSON-LD가 이 검색어를
   정확히 답하는지 점검하고, 필요하면 **의도별 랜딩 섹션/도움말 페이지**를 얹는다(SPA라 index.html
   정적 메타 + 서버 주입이 크롤러 도달의 조건 — analytics.ts:140-146 주석 참조).
2. **게시판 롱테일 Q&A (informational 검색어)** — "SCHD 배당 얼마", "JEPI 월배당 재투자", "배당
   1000만원 만들려면", "FIRE 배당 필요 자금". 이 글들의 강점은 **실제 계산 결과(시뮬 첨부)** 가
   붙는다는 것 — 텍스트만 있는 블로그와 다르다. `has_sim=true` 글이 곧 SEO 콘텐츠 + 도구 유입구.
3. **포트폴리오 갤러리 (UGC 표면)** — 각 시나리오가 색인 가능한 상세 페이지. "SCHD JEPI 포트폴리오",
   "고배당 ETF 조합" 검색을 흡수한다. ⚠ 공유 파라미터(`?s=`)로 canonical이 오염되지 않게 이미
   클린 canonical로 고정돼 있다(analytics.ts:122-138) — 이 규율을 깨지 말 것.

| | |
|---|---|
| 비용 | 낮음(작성 시간) / 인프라는 이미 매몰비용 |
| 난이도 | 중(꾸준함이 관건, 즉효성 없음 — 색인·랭킹은 수 주~수 개월) |
| 예상효과 | **높음·복리**. 한 번 랭크되면 지속 유입. 이 제품의 장기 엔진 |
| GA4 검증 | organic-search 세션 → 활성화율. Search Console/서치어드바이저로 쿼리별 노출·클릭·순위 |

**선행 과제**: **도메인 확정**(현재 `snowball-income.vercel.app` placeholder). 색인·백링크가 임시
도메인에 쌓이면 이전 시 권위가 리셋된다. **콘텐츠 대량 발행 전에 도메인을 못 박아야 한다**(§6 리스크).

### 2-B. 공유 바이럴 — *2순위(제품 내장, 거의 무비용)*

`/api/og` 카드와 lz-string/`?s=` 공유 링크가 이미 있다. 관건은 **"언제·무엇을 공유하고 싶게
만드는가"**.

- **공유 훅 = "내 배당 목표 도달 시점/나이"**. 단순 결과가 아니라 **개인화된 목표 서사**를 카드에
  담는다. 예: *"월배당 300만원 도달 — 2038년 (내 나이 45세)"*. 이건 계산기 스크린샷이 아니라
  **자기 이야기**라 공유·저장 동기가 생긴다(로드맵 #1 목표 달성 시스템과 직결).
- **공유 유도 타이밍**: 아무 때나 "공유하기" 버튼을 상시 노출하는 것보다, **결과가 인상적일 때**
  (목표 달성 배지가 떴을 때, 시뮬을 완주했을 때) 맥락 프롬프트를 띄우는 게 전환이 높다(가설 —
  `scenario_shared` / `simulation_result_view` 비율로 검증).
- **커뮤니티 글쓰기 = 공유의 저마찰 버전**. "공유 링크 복사"는 붙여넣을 곳이 필요하지만, 갤러리에
  올리면 그 자체가 우리 SEO 표면이 된다. **"결과를 갤러리에 올리기" CTA**를 결과 화면에 배치하면
  공유 루프와 콘텐츠 공급이 동시에 돈다.

| | |
|---|---|
| 비용 | 매우 낮음(기존 인프라 활용) |
| 난이도 | 낮음~중(카피·타이밍·목표 카드 디자인) |
| 예상효과 | 중~높음. 활성 유저가 늘수록 복리(K인자). 단 콜드스타트엔 모수가 작아 초기 효과 제한 |
| GA4 검증 | `scenario_shared{share_method}` / WAS, 공유 링크(`?s=`) 유입 세션의 활성화율 |

### 2-C. 커뮤니티 시딩 — *3순위지만 #1의 잠금 해제(콜드스타트의 핵심)*

공개 글 3건 → 다수로 늘리는 것이 **SEO(§2-A)와 바이럴(§2-B) 둘 다의 전제**다. 두 갈래로 간다.

1. **운영자 직접 작성 (내부 시딩)** — ⚠ **합성 페르소나·가짜 유저 생성은 폐기된 결정이다**
   (decisions.md / MEMORY: 투명한 직접 작성으로 전환). 운영자가 **자기 명의로** 실제 시뮬 결과를 붙인
   글을 쓴다: 인사이트("SCHD vs JEPI 20년 재투자 돌려봄"), 질문&고민 유도, 공지. 목표는 **빈 방을
   채워 첫 방문자가 "죽은 서비스"로 느끼지 않게** + **롱테일 검색어에 실 콘텐츠를 심는 것**.
   글 하나하나가 §2-A의 콘텐츠 축 2·3에 그대로 기여한다.
2. **외부 커뮤니티 진출 (referral 획득)** — 우리가 답을 가진 질문이 이미 올라오는 곳에 **도구를
   맥락에 맞게** 소개한다:
   - **네이버 카페**(배당·미국주식·FIRE 카페), **디시 주식 갤러리**, **레딧 한국 투자 서브**,
     **클리앙/뽐뿌 재테크 게시판** 등.
   - 방식은 **스팸이 아니라 가치 제공**: "월배당 목표 도달 시점 궁금하다"는 질문에 **실제 시뮬 결과
     링크(`?s=`)** 로 답한다. 도구 링크가 아니라 **그 사람 질문에 대한 계산 결과**를 준다 → 자연 클릭.
   - ⚠ 각 커뮤니티 규칙 준수(홍보 금지 규정), **과도·반복 노출은 역효과**(§6). 저품질 시딩은 브랜드를
     깎는다 — 양보다 **맥락 적합성**.

| | |
|---|---|
| 비용 | 낮음(시간·꾸준함) |
| 난이도 | 중(외부 커뮤니티 규칙·톤 관리, 지속성) |
| 예상효과 | **초기 최고**. 콜드스타트를 깨는 유일한 지렛대. referral 즉효 + 시드 글이 SEO로 복리 |
| GA4 검증 | referral source별 세션·활성화율, `community_post_published` 누적, 갤러리→시뮬 퍼널 |

### 2-D. 제품 내 루프 — *리텐션(획득한 유입을 새지 않게)*

유입을 아무리 늘려도 한 번 쓰고 떠나면 WAS가 안 큰다. 로드맵 Phase 2가 곧 리텐션 엔진이다.

- **목표 추적 + 재방문 훅**(로드맵 #1): 목표를 저장하면 "달성률·예상 달성일"을 보러 다시 온다.
  `return_visit`·`is_returning` 코호트로 저장 유저의 리텐션 리프트를 측정(ga4-plan Q4).
- **배당 캘린더**(로드맵 #4): "이번 달 예상 입금" — 매달 방문 동기.
- **저장·클라우드 동기화**(이미 있음): 계정 유저는 기기 간 이어서 쓴다 → 재방문·잠금 효과.
- **커뮤니티 참여**: 좋아요·댓글 알림이 없더라도, 내 글에 반응이 쌓이면 돌아온다.
- ⚠ 리텐션 기능은 **유입이 어느 정도 흐른 뒤** 레버리지가 생긴다. 콜드스타트 단계(D0~90)에서는
  §2-A/C(공급)와 §2-B(공유)가 우선이고, 리텐션 기능은 Phase 2 일정에 맞춰 병행한다.

## 3. 콜드스타트 90일 플랜

> 순서가 중요하다. **인프라·계측 → 콘텐츠 공급 → 확산 → 측정·반복**. 각 마일스톤은 앞 단계에 의존한다.

### M1 (Week 1–2) — 토대: "측정과 색인이 가능한 상태" *← 첫 마일스톤*
- [ ] **도메인 확정**(placeholder 탈출) — 이후 모든 색인·백링크가 여기 쌓인다. **콘텐츠 발행보다 먼저.**
- [ ] **Search Console + 네이버 서치어드바이저** 등록, 소유권 확인, `/api/sitemap`·`robots.txt` 제출.
- [ ] **GA4 콘솔 세팅 완료** — ga4-plan §8 체크리스트(Custom Dimensions·User Properties·Key Events
      등록, DebugView로 발화 확인). 이게 돼야 이후 실험을 측정한다.
- [ ] 메인 도구 랜딩 SEO 점검(§2-A 축1): title/H1/JSON-LD/OG가 "배당 계산기" 의도를 정확히 답하는가.
- [ ] **운영자 시드 글 10건**(§2-C 축1) — 각 글에 실제 시뮬 첨부. 롱테일 검색어를 의식해 제목 작성.

### M2 (Week 3–6) — 공급: "색인할 것을 쌓는다"
- [ ] 시드 글 누적 30건 목표(주 3~5건 페이스). 인사이트·Q&A 중심, `has_sim=true` 우선.
- [ ] 결과 화면에 **"갤러리에 올리기" + "공유하기"** CTA 정비(§2-B) — 공유 훅 카피를 "목표 도달
      시점" 서사로.
- [ ] `/api/og` 카드가 **개인화된 목표 서사**를 담도록 카피/레이아웃 점검(로드맵 #1 연계).
- [ ] Search Console에서 **첫 색인·첫 노출 쿼리** 확인 → 실수요 검색어를 글 주제로 역류.

## 4. 측정·실험 — 채널을 어떻게 죽이고 키우나

- **판정 지표는 채널마다 다르다.**
  - SEO: Search Console **노출·클릭·평균순위**(GA4보다 선행 신호) + organic 세션의 **활성화율**.
  - 공유: `scenario_shared` 발생량 + `?s=` 유입 세션의 활성화율(공유가 실제 방문을 데려오나).
  - 외부 커뮤니티: **referral source별** 세션·활성화율(§2-C) — 유입만 많고 활성화율이 낮으면
    잘못된 곳에 홍보한 것.
- **죽이기/키우기 규칙**: 유입은 있는데 **활성화율이 사이트 평균 이하**면 그 채널의 검색어·문구·타깃이
  제품과 안 맞는 것 → 문구 교정 후에도 안 오르면 **중단**. 활성화율이 평균 이상이면 **더 투자**.
- **실험 단위**: 한 번에 하나만 바꾼다(콘텐츠 축·공유 카피·진출 커뮤니티). 콜드스타트라 모수가
  작으므로 **통계적 유의보다 방향성**을 본다 — 미세 최적화(버튼 색 A/B 등)는 아직 이르다.
- ⚠ **데이터를 지어내지 않는다.** 모수가 0에 가까운 구간의 지표는 "관측 불가"로 보고하고, 판단은
  선행 신호(Search Console 노출, referral 클릭)로 대체한다.

## 5. 이 제품 특유의 유입 훅 (다른 계산기와의 차별화)

1. **"내 배당 목표 도달 나이/시점" 공유 카드** — 경쟁 계산기는 "현재 배당액"을 답하지만 우리는
   **시간축**을 답한다. *"월배당 300만원 — 2038년, 45세"* 는 계산 결과가 아니라 **개인의 서사**라
   공유·저장·재방문을 유발한다(로드맵 #1과 결합).
2. **시뮬 결과가 첨부된 커뮤니티 글** — 텍스트뿐인 블로그·카페 답변과 달리, 우리 글은 **실제 계산이
   붙는다**. 롱테일 검색어("SCHD 배당 얼마")에 **재현 가능한 계산 결과**를 심어 SEO 신뢰도·체류를 만든다.
3. **결과 링크로 대화에 답하기** — 외부 커뮤니티 질문에 "이렇게 계산돼요" 하며 `?s=` 링크를 던지면,
   그 링크가 **질문자 맞춤 결과**라 클릭률이 일반 홍보 링크와 다르다. 도구를 파는 게 아니라 **답을 준다**.
4. **UGC 갤러리 = 자가 성장 SEO 표면** — 유저가 올린 포트폴리오 시나리오가 색인 페이지가 되어,
   운영자가 안 쓴 검색어까지 흡수한다(커뮤니티가 클수록 SEO가 스스로 자란다 — 로드맵 철학).

## 7. 요약 — 채널 우선순위와 근거

1. **SEO/콘텐츠**(복리 자산, 인프라 이미 매몰비용) — 장기 엔진. 단 **콘텐츠 공급이 전제**.
2. **커뮤니티 시딩**(내부 직접 + 외부 referral) — **콜드스타트를 깨는 유일한 지렛대**이자 #1의 연료.
   초기 효과는 여기서 가장 크다.
3. **공유 바이럴**(제품 내장, 무비용) — "목표 도달 시점" 훅으로 활성 유저가 늘수록 복리.

> **한 줄 요약**: *공급(시딩) → 색인(SEO) → 확산(공유·referral)* 순서로 돌린다. 콜드스타트 구간의
> 첫 지렛대는 **운영자 직접 시딩 + 외부 커뮤니티 질문 응답**이고, 그 콘텐츠가 방금 깐 ISR 인프라를
> 타고 **네이버·다음·구글에 색인**되며, 활성 유저의 **"내 목표 도달 시점" 공유**가 복리를 얹는다.
