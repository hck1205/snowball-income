# Snowball Income — 서비스 소개 자료 (네이버 로그인 사전 검수 소명용)

---

## 1. 서비스 개요

**서비스명**: Snowball Income
**서비스 URL**: https://snowball-income.vercel.app
**한 줄 소개**: 사용자가 직접 입력한 가정을 계산해 보여주는 **배당 재투자 시뮬레이션 계산기(웹 도구)**

### 무엇을 하는 서비스인가

사용자가 아래 값을 **직접 입력**하면, 그 입력을 수식에 넣어 장기 결과를 계산해 그래프와 표로 보여줍니다.

| 사용자가 입력하는 것 | 예시 |
|---|---|
| 보유(또는 가정할) 종목과 비중 | SCHD 60% / JEPI 40% |
| 배당률·배당 성장률·지급 주기 | 3.5% / 연 8% / 분기 |
| 초기 투자금 · 월 적립액 | 1,000만원 / 월 50만원 |
| 투자 기간 · 세율 · 재투자 여부 | 20년 / 15.4% / 재투자 O |
| 목표 월 배당금 | 월 200만원 |

**계산 결과로 보여주는 것**: 연도별 자산·배당금 추이, 목표 월 배당 도달 예상 시점, 세후 현금흐름, 전량 매도 시 양도세 추정치, 월별 배당 캘린더.

**본질은 이자·복리 계산기와 같습니다.** 사용자가 넣은 숫자를 정해진 수식으로 계산해 되돌려주는 도구입니다.

## 2. 유료 서비스 여부 — **전혀 없음**

- **모든 기능이 완전 무료**입니다. 유료 회원제·구독제·부분 유료(프리미엄) 기능이 **하나도 없습니다.**
- **결제 수단을 일절 연동하지 않았습니다.** 결제 모듈(PG사·카드·간편결제·인앱결제)이 서비스에 존재하지 않으며, 결제·구독·환불과 관련한 화면 자체가 없습니다.
- 유료 전환 계획이 없으며, 사용자에게 금전을 수취하는 어떠한 경로도 두고 있지 않습니다.
- **광고도 게재하지 않습니다.**

> 확인 방법: 서비스 전체를 이용해 보셔도 결제·구독·요금제 안내 화면이 존재하지 않습니다.

---

## 3. 금융투자자문업 해당 여부 — **해당하지 않음**

「자본시장과 금융투자업에 관한 법률」상 **투자자문업**은 금융투자상품의 가치나 투자판단에 관하여 **자문에 응하는 것**을 영업으로 하는 것을 말합니다. 본 서비스는 아래와 같은 이유로 이에 해당하지 않습니다.

### 3-1. 종목을 추천하거나 매매를 권유하지 않습니다
- 서비스는 **어떤 종목도 추천하지 않습니다.** "이 종목을 사세요/파세요" 같은 매매 권유, 매수·매도 시점 제시, 목표가 제시 기능이 **없습니다.**
- 종목·비중·수익률 등 **모든 값은 사용자가 직접 입력**합니다. 서비스는 그 입력을 계산할 뿐, 무엇을 사야 하는지 판단해 주지 않습니다.
- 제공되는 "프리셋"은 사용자의 입력 편의를 위한 **예시 묶음**이며, 추천이 아니라 출발점입니다. 사용자는 이를 자유롭게 수정·삭제합니다.

### 3-2. 자문·상담·일임 행위가 없습니다
- 1:1 상담, 자문 응대, 포트폴리오 진단·처방, 매매 대행(일임) 기능이 **일절 없습니다.**
- 사람이 개입하는 어떠한 자문 창구도 운영하지 않습니다.

### 3-3. 수익을 보장하거나 예측을 단정하지 않습니다
- 결과는 **사용자가 입력한 가정에 따른 산술적 계산값**이며, 미래 수익을 예측하거나 보장하는 것이 아닙니다.
- 실시간 시세를 제공하지 않으며, 이를 서비스 내에 명시하고 있습니다.

### 3-4. 대가를 받지 않습니다
- 투자자문업은 **영업으로서 대가를 받는 것**을 요건으로 하나, 본 서비스는 위 2항과 같이 **어떠한 금전도 수취하지 않습니다.**

### 3-5. 서비스 내 고지 문구 (실제 노출 중)

서비스 곳곳에 아래 문구가 이미 노출되고 있습니다.

- 메인 화면 하단:
  > "…자문이 아니며, 참고용으로만 사용해 주세요. 실제 수익률과 배당은 시장 상황에 따라 달라집니다."
- 계산 조건 변경 안내:
  > "이 앱은 입력한 가정을 그대로 계산해 보여주는 시뮬레이터입니다. **투자 자문이 아니니 참고용으로만** 사용해 주세요."
- 종목 데이터 화면:
  > "주의: 실시간 데이터가 아니기 때문에 실제 데이터와 다를 수 있습니다. 참고용으로만 사용해 주세요."
- 결과 리포트(PDF) 하단:
  > "이 리포트는 투자 조언이 아닙니다. … 세법과 환율은 변동합니다. 과거 데이터가 미래를 보장하지 않습니다."

## 4. 네이버 로그인을 사용하는 이유와 범위

- **로그인 없이도 모든 계산 기능을 이용할 수 있습니다.** 로그인은 선택 사항입니다.
- 로그인은 아래 용도로만 사용합니다.
  1. 사용자가 만든 시뮬레이션 설정을 **여러 기기에서 이어서 보기** 위한 저장·동기화
  2. 커뮤니티(자유게시판·포트폴리오 공유) 글 작성 시 **작성자 식별**
- 네이버로부터 제공받는 정보는 서비스 이용에 필요한 최소 범위로 한정합니다.
- **네이버 로그인 과정에서 서비스 고유의 별도 비밀번호를 요구하지 않습니다.**

## 5. 운영 주체

- 개인 개발자가 만든 **비영리 성격의 개인 프로젝트**입니다.
- 법인이 아니며, 수익 모델이 없습니다.

## 부록 B. 로그인 동작 화면 캡처 순서

네이버가 요구하는 것은 **"가입하지 않은 신규 이용자"** 기준의 전체 흐름입니다. 아래 순서대로 캡처하세요.

**사전 준비**
1. 네이버 개발자센터 > 내 애플리케이션 > 멤버관리에서 테스트 ID가 등록돼 있는지 확인
2. 브라우저를 **시크릿 창**으로 열어 기존 세션이 없는 상태로 시작
3. 해당 네이버 계정이 이 서비스에 **이미 가입돼 있다면** 먼저 탈퇴(서비스 내 회원 탈퇴 기능 사용)해야 "신규 가입" 흐름이 나옵니다 — 이게 빠지면 동의 화면이 뜨지 않아 또 반려될 수 있습니다

**캡처 순서 (①~⑤ 전부 필요)**
1. **로그인 진입 전 화면** — 서비스 메인, 로그인 버튼이 보이는 상태
2. **로그인 수단 선택 화면** — 네이버 로그인 버튼이 명확히 보이는 화면
3. **네이버 로그인 페이지** — 네이버 계정 입력 화면
4. **정보 제공 동의 화면** — 제공 항목과 "동의하기" 버튼이 보이는 화면 ← **가장 중요. 이게 빠져서 반려됐습니다**
5. **로그인/가입 완료 후 화면** — 서비스로 돌아와 로그인된 상태(닉네임·프로필이 보이는 헤더 등)

**주의**
- 5장 모두 **연속된 하나의 흐름**임이 드러나야 합니다
- 캡처에 개인정보(실명·이메일 전체 등)가 노출되지 않도록 필요시 가리세요
- 파일은 5MB 이하, 최대 5개까지 업로드 가능합니다
