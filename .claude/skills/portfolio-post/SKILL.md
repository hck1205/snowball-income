# 포폴 갤러리 글 작성

포폴 갤러리 글을 쓸 때 아래 규칙과 구조를 따른다. 깊은 작성/여러 버전 비교가 필요하면
`portfolio-post-writer` 서브에이전트에 위임해도 되지만, 규칙은 동일하다.

## 절대 규칙 (하나라도 어기면 실패)

1. **1인칭 금지.** "저는 / 제가 / 내가 / 나" 금지. 작성자 고백이 아니라 **방법을 소개·연구하는
   형태**로 쓴다("이런 구성도 있다", "~하는 방식이다").
2. **담백한 톤.** 과장·낚시성 구어체 금지. 예: "언제 녹여 먹죠", "딸깍", "멍했습니다" 류 금지.
3. **결과 숫자 본문 금지.** 15년 뒤 월 배당·누적 배당·최종 자산 등 **시뮬레이션 산출값**은 넣지
   않는다 → "결과는 첨부한 시뮬레이션에서 확인". 단 **구성값**(비율·종목별 배당률·가중평균
   배당률/성장률)은 허용.
4. **"눈덩이 / 스노우볼" 표현 금지.** 대신 **복리 · 시간 · 재투자** 를 적극적으로 쓴다.
5. **투자 자문 아님 고지 필수** + 수익 보장·단정 표현 금지. 전략의 **단점도 정직하게** 짚는다.
6. 한국어.

## 구조

1. 도입(통념과 대비되는 맥락) → 2. 포트폴리오(비율 + 종목별 한 줄 역할) →
3. 가중평균 배당률·배당성장률 → 4. 배분 이유 → 5. 조건(초기금·월적립·기간·재투자·세율) →
6. "결과는 첨부 시뮬레이션 참고" + 그래프에서 볼 포인트 1~2개 → 7. 짚어둘 점(정직한 트레이드오프,
복리·시간·재투자 언어) → 8. 각자 조건으로 돌려보라는 권유 → 9. Disclaimer.

## 숫자

- 종목별 배당률·기대총수익률은 `shared/constants/presets/` 에서 확인해 쓴다(지어내지 않는다).
- 가중평균은 비율×배당률로 계산 가능(구성 설명값).
- 결과 숫자가 꼭 필요하면 "시뮬레이터로 돌려 첨부하라"고 안내한다.

## 산출물

제목 후보 1~3개 + 바로 붙여넣을 수 있는 본문 완성본.
