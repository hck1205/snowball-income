# 팀 지식 기반 (Team Knowledge Base)

> 에이전트 팀의 **누적 학습 저장소**. 세션이 끝나도 남고, 다음 작업의 출발점이 된다.
> 모든 에이전트는 **작업 시작 전 이 INDEX를 읽고**, 자기 작업과 관련된 파일을 확인한다.
> 작업 종료 시 "코드만 봐서는 알 수 없는" 교훈을 얻었으면 해당 파일에 추가한다.

| 파일 | 내용 | 언제 읽나 |
|---|---|---|
| [decisions.md](decisions.md) | 확정된 제품·디자인 결정과 근거 — 되돌리려면 사용자 승인 필요 | 모든 구현·설계 작업 전 |
| [pitfalls.md](pitfalls.md) | 실제로 겪은 함정·사고·기술 제약 — 같은 실수 반복 방지 | 모든 작업 전 (특히 자기 도메인 항목) |
| [project-map.md](project-map.md) | CLAUDE.md 이후 추가된 구조 지도 (테마 시스템, 커뮤니티 등) | 처음 맡는 영역을 만질 때 |
| [retro.md](retro.md) | 협업 프로세스 회고 — 무엇이 잘 작동했고 무엇이 사고를 냈나 | orchestrator가 미션 설계 시 |
| [user-profile.md](user-profile.md) | 사용자의 취향·소통 방식 — 제안의 방향을 정할 때 | 설계·제안 작업 전 |
| [roadmap.md](roadmap.md) | 제품 로드맵(우선순위·Phase) + 코드 연결점·의존성 | 새 기능 제안·설계 시 (정식 문서는 docs/ROADMAP.md) |
| [architecture.md](architecture.md) | 로드맵 기능의 확장지점 맵 + 공통 토대 2건(시계열 스냅샷/티커 확장) + 확장 관례 | 로드맵 기능 착수·설계 시 (정식 문서는 docs/ARCHITECTURE.md) |

## 기록 규칙 (전 에이전트 공통)

- 형식: `- [YYYY-MM-DD][도메인] 교훈 한 문장 — 근거 path:line 또는 사건 요약`
- **중복 금지**: 추가 전에 파일 내 검색. 이미 있으면 그 항목을 갱신.
- **사실이 바뀌면 수정/삭제** — 지식 기반이 낡으면 독이 된다.
- CLAUDE.md·코드 주석이 이미 말하는 것은 넣지 않는다. **여기엔 "겪어봐야 아는 것"만.**
- 파일이 ~120줄을 넘으면 orchestrator가 병합·정리한다.
